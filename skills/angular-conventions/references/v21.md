# Angular v21

Released 2025-11-19; currently in LTS (active support ended when v22 shipped). The rest of SKILL.md applies; this file states what v21 does NOT yet have that the skill's forward idioms assume, plus its v21-specific changes. Load it only if the workspace is on v21. Fact-checked against angular.dev.

## Runtime

- Node.js `^20.19.0 || ^22.12.0 || ^24.0.0`, TypeScript `>=5.9.0 <6.0.0`. RxJS `^6.5.3 || ^7.4.0`.
- Support: LTS (critical + security fixes only).

## New or default in v21

- **Zoneless is the default for new apps** - `zone.js` is no longer bundled by default. Bootstrap `provideZonelessChangeDetection()` (stable since v20.2).
- **Signal Forms ships - but experimental.** `form()` from `@angular/forms/signals` with `validate` / `validateAsync` / `validateHttp` / `validateStandardSchema` all landed in v21.0 as an experimental library. Usable, but version-tag every snippet and expect API churn; it does not stabilize until v22.
- **`@angular/aria`** ships as developer preview (v21 launched 8 patterns: Accordion, Combobox, Grid, Listbox, Menu, Tabs, Toolbar, Tree).
- Host-binding type-checking is on by default (v20 required a tsconfig opt-in); v21 also ships `ngClass` / `ngStyle` -> plain `class` / `style` binding migrations.
- `ViewEncapsulation.ExperimentalIsolatedShadowDom` (experimental) - ShadowDom plus a hard block on external styles leaking in.

## Still experimental / NOT yet stable in v21 - do not treat as the default

- The resource family (`resource`, `rxResource`, `httpResource`) is still experimental here - it stabilizes in v22. Pin to the installed version before leaning on it.

## Hydration wiring

- Incremental hydration is opt-in via `provideClientHydration(withIncrementalHydration())` - it is NOT default until v22. It auto-enables event replay, so do not also add `withEventReplay()`.

## Material / CDK (see `angular-material`)

- `FocusTrap` construction and CDK `Overlay` internals changed in v21 - the breaking-change list is `angular-material`'s `references/versions.md`. The `cdkTrapFocus` directive is unaffected - prefer it; the `Overlay` service API you call is unchanged.
