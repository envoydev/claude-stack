# Angular v20

Released 2025-05-28; currently in LTS (ends 2026-11-28). The rest of SKILL.md applies; this file states what v20 does NOT yet have that the skill's forward idioms assume, plus its v20-specific renames. Load it only if the workspace is on v20. Fact-checked against angular.dev.

## Runtime

- Node.js `^20.19.0 || ^22.12.0 || ^24.0.0`; TypeScript `>=5.8.0 <5.9.0` on 20.0/20.1, widened to `<6.0.0` on 20.2/20.3. RxJS `^6.5.3 || ^7.4.0`.
- Support: LTS (critical + security fixes only).

## New or stable in v20

- `effect()`, `linkedSignal`, and `toSignal` reached stable.
- **Incremental hydration reached stable** - opt in with `provideClientHydration(withIncrementalHydration())` (auto-enables event replay; do not also add `withEventReplay()`). It is NOT on by default here (that is v22).
- **Zoneless reached stable in v20.2** (developer preview earlier in v20) - but it is NOT the default for new apps until v21. Bootstrap `provideZonelessChangeDetection()` explicitly.
- Style-guide change: from v20 the CLI stops generating `.component` / `.service` / `.directive` type suffixes by default - the house drop-suffix naming in `references/angular-style.md` applies here as written. Keep the suffixes only where the project's own config or its `docs/CODE-STYLE.md` says to - the project override wins; either way migrate organically, never mass-rename.

## v20 API renames - version-tag any snippet

- `resource`'s `request` option was renamed `params`, and `rxResource`'s `loader` was renamed `stream`. Code written for v19 uses the old names.

## Still experimental / NOT yet available in v20 - do not reach for these

- The resource family (`resource`, `rxResource`, `httpResource`) is still experimental (stabilizes v22).
- **Signal Forms does not exist yet** - it ships (experimental) in v21. Use typed reactive forms (`FormGroup<T>`) here.
- `@angular/aria` does not exist yet - it ships (developer preview) in v21.

## Material / CDK (see `angular-material`)

- The `matButton` attribute replaces the `mat-button` family (`matButton`, `matButton="elevated|filled|outlined|tonal"`), and Material's raw tokens were renamed `--mdc-*` -> `--mat-*`. Run `ng generate @angular/material:token-rename` on upgrade; the old button directives are deprecated, not deleted.
