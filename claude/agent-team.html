Agent roster · built · one seat, one expert

Every domain owns its vertical.
One hand designs it, one builds it, one verifies it.

The team is a matrix: a project architect that scopes and directs, then for each stack three hands - a solution designer that owns the architecture, plan and test strategy and splits the work into parallel tasks; several implementers that each build one part, code and its tests; and a verifier that, once every part lands, checks the whole against the plan and code quality and hands a punch-list back until it matches. No agent is asked to be good at both Angular and C# - each seat is a single deep role.

03

Built

Settled

  • Per-domain vertical, fanned out. The designer decomposes the work into parallel tasks with contracts; several implementer instances build them at once; the verifier gates the assembled whole and loops a punch-list back. Five stack-agnostic pipeline agents dissolve into it; the fan-out is main-session orchestrated (a domain-build skill); the vertical's seats never dispatch agents. The sole nested dispatch in the roster is the two diagnosers calling a cheap read-only evidence-gatherer to keep log volume off the opus seat.
  • No rank words. Seats are named <domain>-solution-designer, <domain>-implementer and <domain>-verifier - stack + function, no senior / middle.
  • Pins. Designers Opus 4.8 / xhigh (top-level design judgment); verifiers Sonnet 5 / xhigh (an independent gate at full effort); implementers Sonnet 5 / medium (build to the contract).
  • Skills rewired. project-scaffold and project-quality-loop were repointed off the retired agents onto the domain seats and domain-build.

Shipped this session

  • Fifteen seats + the domain-build skill created; the five generic agents (implementation-planner, test-planner, code-implementer, test-writer, change-verifier) retired into them.
  • Plus a cross-cutting issue-diagnoser - the debugger, and the bug-side counterpart of a solution designer. It reads logs, errors and screenshots, isolates the root cause, and lays out a decomposed fix plan that feeds the same implementers → verifier loop. It plans; it never writes the fix. Roster is now 24.
  • Agents hardened, skills enriched. The four resolvers and ci-failure-diagnoser now run the superpowers systematic-debugging method; implementers report a DONE / DONE_WITH_CONCERNS / NEEDS_CONTEXT / BLOCKED status and domain-build gained a compaction-safe progress ledger + per-task diff review (from subagent-driven-development). Seven best-practice field guides were distilled into nine house skills (29 additions, private names scrubbed), and the WPF seats now reach dotnet-hosted-services for a companion Windows Service.
  • Two cross-cutting seats added - roster 24 → 26. cross-stack-contract-designer (opus/xhigh, read-only) freezes the shared backend↔frontend contract - DTOs, routes, error envelope, auth, pagination, versioning - before the per-stack designers, so a full-stack feature builds to one agreed seam instead of drifting into a runtime bug. framework-upgrade-planner (opus/xhigh, read-only) turns a version/deprecation event - a .NET or Angular major, an EOL - into an ordered, contracted upgrade plan the implementers and resolvers execute. Both fill gaps the adversarial roster audit confirmed no existing seat covered.
  • Cross-cutting security-auditor added - roster 27 → 28. A read-only, opus/xhigh security seat that audits the posture of a feature or codebase across all five stacks - auth, authorization, input/injection, secrets, config, data exposure - and returns an OWASP/CWE punch-list that routes to the domain implementers to fix and the verifier to confirm. It complements /security-review (diff/PR-scoped) and the security-guidance commit hook by being the deep, posture-scoped, routed audit; it finds and reports, never writes the fix. Claude-only.
  • Verified - npm run lint clean, all 28 frontmatters valid, 4-way parity held.
